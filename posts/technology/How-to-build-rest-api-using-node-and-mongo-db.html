Building a RESTful API with Node.js and MongoDB: A Step-by-Step Guide

A REST API (Representational State Transfer Application Programming Interface) is an architectural style that defines a set of constraints for creating web services. REST allows simple and flexible communication between clients (such as web browsers or mobile apps) and servers. It uses standard HTTP methods (GET, POST, PUT, PATCH, DELETE) for interactions.

6. Create a GitHub Repository

Go to GitHub and log in. Click the "+" icon in the top right corner and select "New repository." Fill in the repository name, description, and other settings. Click "Create repository."

7. Config the git user

In the terminal to configure the username, run: git config --global user.name "username".
To configure the user email, run: git config --global user.email "username@email.com".

8. Link Local Repository to Remote GitHub Repository

Copy the repository URL from GitHub (it looks like https://github.com/username/my-repo.git).
In your terminal, run: git remote add origin repo_url.

9. Push to GitHub

Push your local repository to GitHub: git push -u origin master
