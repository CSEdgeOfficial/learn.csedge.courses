How to set up your local GitHub repository

Git is a distributed version control system that tracks versions of files. It is often used to control source code by programmers collaboratively developing software. Git allows developers to keep track of code changes, collaborate on projects, and manage code efficiently.

Pre-requisites

Before you begin, ensure that you have installed Git on your computer. You can install from the Git's official website (Link)

1. Create a New Directory

Open the terminal(git bash) or command prompt. Navigate to the directory where you want to create your local repository. Run the following command to create a new directory: mkdir my-repo

2. Initialize Git Repository

Inside your newly created directory, initialize a Git repository: git init

3. Create File

Create the files you want to include in your repository (e.g., index.html, main.py, etc.).

4. Add Files to Staging Area

Add your files to the staging area using the following command: git add .

5. Commit Changes

Commit your changes with a descriptive message: git commit -m "Commit Message".

6. Create a GitHub Repository

Go to GitHub and log in. Click the "+" icon in the top right corner and select "New repository." Fill in the repository name, description, and other settings. Click "Create repository."

7. Config the git user

In the terminal to configure the username, run: git config --global user.name "username".
To configure the user email, run: git config --global user.email "username@email.com".

8. Link Local Repository to Remote GitHub Repository

Copy the repository URL from GitHub (it looks like https://github.com/username/my-repo.git).
In your terminal, run: git remote add origin repo_url.

9. Push to GitHub

Push your local repository to GitHub: git push -u origin master
